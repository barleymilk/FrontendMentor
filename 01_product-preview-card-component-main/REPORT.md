# Product Preview Card Component - Solution Report

## 프로젝트 개요

이 프로젝트는 Frontend Mentor의 **Product preview card component** 챌린지를 바탕으로 구현되었습니다. 주어진 디자인을 최대한 정확하게 재현하고 반응형 레이아웃을 적용하는 것이 목표입니다.

## 결과 요약

- 전체 점수: **8.3 / 10**
- 코드 품질: **8.5 / 10**
- 디자인 및 UX: **8.0 / 10**
- 요약: 탄탄하고 의미론적인 제품 카드를 만들었으며, 마크업에 랜드마크와 `lang=` 속성이 사용되었습니다.

## PDF 보고서에서 확인된 주요 사항

### 장점

- `lang="en"` 요소를 사용하고, 데스크톱 버전과 대체 버전을 `<html>` 요소로 구분하여 화면 낭독기와 다양한 뷰포트에 맞춘 반응형 이미지 환경을 제공함.
- `picture`, `source`, `img` 요소를 사용하여 대형 화면에서 적절한 이미지를 제공하고 접근성을 위해 텍스트를 유지함.
- 버튼 내부에 장바구니 아이콘과 `Add to Cart` 텍스트를 함께 사용하여 시각적 제어 요소 구성이 명확함.

### 개선 권장 사항

- `Add to Cart` 버튼에 명시적인 `:focus` 또는 `:focus-visible` 스타일을 추가하여 키보드 사용자가 포커스 위치를 분명히 알 수 있도록 해야 함.
- 독립형 페이지 구조에서 주요 제목은 `<h1>`을 사용하여 문서 계층 구조를 명확히 하는 것이 좋음.
- 버튼 안 장바구니 아이콘은 장식 요소로 처리해야 하므로 `alt=""` 및 `aria-hidden="true"`를 추가하여 화면 읽기 프로그램이 불필요한 텍스트를 읽지 않게 해야 함.

## 구현 내용

- 모바일 및 데스크톱 화면에 대응하는 반응형 카드 레이아웃 구현
- 제품 이미지, 제목, 설명, 가격, 구매 버튼을 포함한 카드 구성
- `normalize.css`로 브라우저 기본 스타일 초기화
- `style-guide.md` 기준 색상, 폰트, 간격 스타일 반영
- 시맨틱 HTML 구조를 사용하여 콘텐츠 의미 전달

## 접근성 및 코드 품질 검토

- `lang="en"` 설정으로 페이지 언어를 명시함
- `picture`/`source`/`img` 구성을 통해 반응형 이미지 제공 권장
- 버튼 포커스 스타일 보강으로 접근성 향상 가능
- 주요 제목 태그 계층 구조를 개선할 여지가 있음
- 아이콘 이미지에 `alt`, `aria-hidden` 속성 추가 필요

## 테스트

- 이미지 대체 텍스트 및 접근성 속성 확인
- 시맨틱 HTML 구조 검토
